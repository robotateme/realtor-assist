# Project Description

## Что это

`Realtor Assist` сейчас развивается как помощник юриста и сборщик лидов для его компании.

Это backend-платформа, которая должна:

- принимать входящие обращения из мессенджеров
- регистрировать и сохранять лидов
- сопровождать первичный диалог
- помогать квалифицировать запрос клиента
- готовить почву для дальнейшей юридической консультации
- подключать AI-сценарии для ответов, маршрутизации и обработки контекста

## Основной продуктовый сценарий

Пользователь пишет в Telegram-бота.

Система:

1. принимает webhook
2. идентифицирует или регистрирует пользователя
3. сохраняет данные лида
4. продолжает диалог по заранее определённому или AI-assisted сценарию
5. передаёт собранный контекст юристу или во внутренний workflow компании

## Что уже подготовлено в проекте

- Telegram webhook infrastructure
- очереди и фоновые задачи через Redis + Horizon
- outbox pattern для доменных событий
- базовые доменные сущности для клиентов и диалоговых сессий
- интеграция с Ollama для AI-функциональности
- prompt/rendering инфраструктура для LLM-слоя

## Что ещё предстоит развивать

- полноценную lead qualification логику
- сценарии первичного интервью клиента
- привязку юридических услуг и типов обращений
- CRM-поток для обработки заявок
- отчётность и аналитику по лидам
- операторский интерфейс или внутренний кабинет

## Роль AI в системе

AI-слой нужен не как абстрактный чат, а как прикладной инструмент:

- для первичного разбора запроса клиента
- для маршрутизации обращений
- для генерации черновиков ответов
- для подготовки структурированного контекста для юриста
- для помощи в follow-up и lead nurturing

## Текущее состояние

Проект находится на инфраструктурной стадии: фундамент уже собран, но продуктовые сценарии ещё требуют развития. Из подтверждённого пользовательского потока сейчас реализована базовая регистрация Telegram-пользователя по команде `/start`.
